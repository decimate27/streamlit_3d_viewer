🔍 3D Viewer 로그 확인 가이드

1. 브라우저 콘솔 열기

Windows/Linux:

Chrome/Edge: F12 또는 Ctrl + Shift + I
Firefox: F12 또는 Ctrl + Shift + K

Mac:

Chrome/Edge: Cmd + Option + I
Firefox: Cmd + Option + K
Safari: Cmd + Option + C
💡 또는 웹페이지에서 마우스 우클릭 → "검사(Inspect)" 선택

2. Console 탭에서 확인할 로그

✅ 정상 로그 (성공)

Three.js version: 128 Starting model load... Textures loaded: ['texture1.jpg', 'texture2.png'] Total textures: 2 - texture1.jpg: Loaded successfully - texture2.png: Loaded successfully Loading MTL... Material HeadMaterial -> Texture texture1.jpg ✅ Texture applied: texture1.jpg Material BodyMaterial -> Texture texture2.png ✅ Texture applied: texture2.png Materials loaded Loading OBJ... === Final Material Check === Mesh uses material: HeadMaterial -> Has texture: texture1.jpg Mesh uses material: BodyMaterial -> Has texture: texture2.png ========================= Model loaded successfully

❌ 문제 있는 로그 (에러)

Material BodyMaterial -> Texture body_texture.jpg ⚠️ Texture not found: body_texture.jpg Available textures: ['texture1.jpg', 'texture2.png']

// 콘솔에서 직접 입력하여 특정 로그만 보기

4. 실시간 디버깅 명령어

콘솔에서 직접 입력하여 현재 상태 확인:

// 현재 씬의 모든 메시 확인
scene.children.forEach(obj => console.log(obj))

// 모델의 재질 확인
if (model) {
  model.traverse(child => {
    if (child.isMesh) {
      console.log('Mesh:', child.name, 'Material:', child.material.name);
    }
  });
}

// 로드된 텍스처 확인
console.log('Loaded textures:', Object.keys(textures))

5. 문제 해결 체크리스트

텍스처가 안 보일 때:

  • ✅ "Textures loaded" 로그에 모든 텍스처 파일이 있는지 확인
  • ✅ "Texture not found" 경고가 있는지 확인
  • ✅ MTL 파일의 텍스처 이름과 실제 파일명이 일치하는지 확인
  • ✅ "Final Material Check"에서 각 Material이 올바른 텍스처를 가지고 있는지 확인

모델이 안 보일 때:

  • ✅ "Model loading error" 메시지 확인
  • ✅ OBJ 파일 파싱 에러 확인
  • ✅ Three.js 버전 확인 (현재: 0.128.0)
